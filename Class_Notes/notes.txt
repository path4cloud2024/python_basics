    --> ✅ Object and Dynamic typing
        - whenever we assign some value to any variable then internally, Python creating an identifier for that and it aims a portion of memory, which is an datatype obejct containing that value. e.g.
        x = 10
        here, x is the identifier which contains memory address where value is stored and refers to integer object containing value 10.

        - we can change the any identifier to any object in Python, called dyanmic typing. Python automatically updates the variable type as needed. This happens because type checking is done at runtime, not before execution (compile time).
        e.g.

        >>> x = 10
        >>> x + 5
        15
        >>> x # it still 10 now
        10
        >>> type(x)
        int
        >>> x = 10.5
        >>> type(x)
        float
        
        ** We must know the built-in keywords or identifier before we assign any obejcts type, else it will result to syntax error.

    --> ✅ Builtin range() function:
        we have 3 version of range functions:
            a) with single argumets, it will give the starting value 0 and ending value upto that defined value but not included.

    --> List, Set, Tuple, dictionary

        List: 
            - Ordered (keeps the order of insertion).
            - Mutable (you can change elements).
            - Allows duplicates.
            - denotes with [ ]
 
        Tuple:
            - Ordered
            - Immutable (cannot change once created).
            - Allows duplicates.
            - Use when data should not change (like fixed config values).
            - not mandatory to enclose with () but to better readability or as best practices, we enclose with ()

        Set:
            - Unordered (no index, order may change).
            - Mutable (can add/remove elements).
            - No duplicates allowed (automatically removes them).
            - Great for checking unique items or membership tests (in is very fast in sets).

        Dictionary:
            - Stores data as key: value pairs.
            - Keys must be unique and immutable (string, number, tuple).
            - Values can be anything.
            - Mutable.

        Key Differences (Summary)
        Feature	    List	    Tuple   	Set	       Dictionary
        Ordered	    ✅ Yes	    ✅ Yes	❌ No	    ✅ Yes (since Python 3.7)
        Mutable	    ✅ Yes	    ❌ No	✅ Yes	    ✅ Yes
        Duplicates	✅ Yes	    ✅ Yes	❌ No	    Keys ❌, Values ✅
        Access by	Index	      Index	    No index	 Key
        Use Case	Store 	      Fixed 	Unique 	    Key–value mappings (lookups)
                    sequence      data      items

        👉 Simple analogy:

        List = Shopping list (you can add/change items, order matters).

        Tuple = Passport info (fixed, cannot change).

        Set = Attendees in a meeting (no duplicates).

        Dictionary = Phonebook (name → number mapping).
